## 0. 前提確認（最初に決める）
- [ ] 対象業務：棚卸（在庫数の実数カウント → 差異確認 → 在庫確定）
- [ ] 入力方式：実数（残数）は手入力（カメラ/スキャン機能は使わない）
- [ ] 端末：PC / タブレット / スマホ（キーボード/テンキー入力のしやすさ重視）
- [ ] デザイン要件：モバイルファースト、chadcnライク（shadcn/ui風）のモダンデザイン
- [ ] 利用者/権限：一般のみ
- [ ] ログイン要件：Cookie による長期ログイン（ログイン維持）を行う
- [ ] 実装方針：DDDで実装し、エンティティ/値オブジェクトにビジネスロジックを持たせる
- [ ] 店舗/倉庫：ロケーション（売場/倉庫/バックヤード）を扱う
- [ ] 在庫の確定方式：棚卸差異を「調整」して確定するか、確定後に履歴を残すか
- [ ] 既存データ連携：商品マスタ/在庫のインポート（CSV/Excel）とエクスポートできる
- [ ] オフライン要件：そもそも接続できない
- [ ] 監査要件：いつ誰が何を変更したか（監査ログ）必須である
